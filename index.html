Projects

Handwriting Neural Network

I made a neural network to predict what letter an image contains using Python. The network achieved a success rate of 88.5% on a test set of 10,000 images. I used libraries like EMNIST, Matplotlib, SKLearn, OpenCV, and NumPy.

The EMNIST library was used to get the dataset of all the handwritten images. It contains 70,000 images of which I used 60,000 to train the network and 10,000 to test it.

Matplotlib was used for visualizing my results. I used something called a confusion matrix to visualize where the network was making mistakes, and Matplotlib helped me convert this array into a matrix I can see.

I used SKLearn to get the multilayer perceptron classifier and the confusion matrix. The multilayer perceptron is the backbone of the program, and I used it because I felt like backpropagation would be a good way to train the network for this specific task.

NumPy was used for arrays becaue I found it easier to work with since I could use functions like reshape. This made it easier to feed the data to the network.

Sports Predictor

I created a sports predictor in Python which uses stats from various leagues and outputs predictions for games. The program achieved a win rate of over 60% when tested on 168 games.

The program looks at the average offensive and defensive stats of the league and then calculates "team strengths" for both of the teams playing by comparing them to the average.

Once the program has the two team strengths each (offense and defense), it compares offense to defense and vice versa and then multiples this by the league average to come up with a predicted score.

I then used a Poisson distribution using the predicted score by looping through all the possible scores and using the predicted score as a mean.

When I started this project, I was feeding it CSVs of league stats manually which was fast but way more work. When I started making V2, however, I used an open source API for SportsReference which scrapes the site and gets all the data I need.

I am currently working on V3 which would use a multilayer perceptron regressor instead of manual calculations. I am planning on training it using the SportsReference API since it has box scores of every game and I will use a small portion of the data to test it as well.

Android Game

A friend and I created a version of pong for Android which we called ReBounce. We made the game using the game engine Unity3D and the language C#.

We did not know many AI concepts at the time we made it so the way we coded the "AI" was just to calculate the difference between the Y coordinates of the CPU paddle and the ball and then move the paddle in that direction. As the difficulty went up, so did the speed and size of the paddle.

For moving the player's paddle, the way we did it was just seeing if the user is touching the top or bottom half of the screen, and moving in that direction accordingly.
